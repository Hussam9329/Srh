## 📖 عن المشروع

**نظام إدارة معهد صرح البنوك (SRH)** هو نظام إلكتروني شامل مصمم خصيصاً لإدارة المعاهد التعليمية. يوفر النظام حلولاً متكاملة لإدارة الطلاب والمدرسين والعمليات المالية والتقارير، مع واجهة عربية أنيقة بتصميم داكن يحتوي على لمسات ذهبية تعكس الطابع الرسمي للمعهد.

---

## ✨ المميزات

### 📊 لوحة التحكم والإحصائيات
1. **لوحة تحكم رئيسية مع إحصائيات حية** - عرض فوري لعدد الطلاب والمدرسين والمبالغ المدفوعة والمسحوبة
2. **رسوم بيانية تفاعلية** - تصور بيانات المعهد بشكل مرئي باستخدام Recharts

### 👨‍🎓 إدارة الطلاب
3. **إدارة كاملة للطلاب** - إضافة، تعديل، حذف، بحث متقدم في قائمة الطلاب
4. **تتبع حالة الطالب** - متابعة نوع الدراسة (حضوري / إلكتروني) والحالة (مستمر / منسحب)
5. **توليد باركود تلقائي** - كل طالب يحصل على باركود فريد (BN-XXXX) تلقائياً

### 👨‍🏫 إدارة المدرسين
6. **إدارة كاملة للمدرسين** - إضافة، تعديل، حذف، تجميع حسب المادة الدراسية
7. **نظام النسب المئوية** - تحديد نسبة المعهد من كل مدرس بمرونة

### 🔗 الربط والعلاقات
8. **نظام ربط الطلاب بالمدرسين** - ربط كل طالب بمدرسين متعددين وإدارة العلاقات

### 💰 الأقساط والمدفوعات
9. **نظام أقساط ومدفوعات متكامل** - تسجيل الأقساط المدفوعة مع تاريخ الدفع والملاحظات
10. **محاسبة المدرسين** - حساب النسب، السحوبات، الأرصدة المتبقية تلقائياً
11. **حسابات تلقائية (نسبة المعهد)** - خصم نسبة المعهد من كل قسط حسب نوع الدورة

### 📄 التقارير
12. **تقارير شاملة قابلة للطباعة** - تقارير الطلاب والمدرسين والتقارير المالية
13. **تصدير تقارير HTML** - تصدير التقارير بتنسيق HTML قابل للطباعة

### 🔐 الأمان والمستخدمين
14. **إدارة مستخدمين مع صلاحيات** - ثلاث مستويات: مدير، مستخدم، مشاهد
15. **نظام مصادقة آمن (bcrypt)** - تشفير كلمات المرور مع دعم الترحيل التلقائي

### 🎨 التصميم والواجهة
16. **واجهة عربية RTL كاملة** - دعم كامل للاتجاه من اليمين لليسار
17. **تصميم داكن أنيق** - خلفية سوداء مع لمسات ذهبية احترافية
18. **بحث ذكي مع Debounce** - بحث فوري مع تحسين الأداء

## 🚀 التثبيت

### 1. نسخ المشروع

```bash
git clone https://github.com/Hussam9329/Srh.git
cd Srh
```

### 2. تثبيت الحزم

```bash
bun install
```

### 3. إعداد المتغيرات البيئية

```bash
cp .env.example .env
```

### 4. تهيئة قاعدة البيانات

```bash
bun run db:push
bun run db:generate
```

### 5. تشغيل المشروع

```bash
bun run dev
```

افتح المتصفح على العنوان: [http://localhost:3000](http://localhost:3000)

## 🗄️ قاعدة البيانات

يستخدم المشروع **Prisma ORM** مع **SQLite** كقاعدة بيانات. يتكون المخطط من 6 جداول:

| الجدول | الوصف |
|--------|-------|
| `students` | بيانات الطلاب (الاسم، نوع الدراسة، الباركود، الحالة) |
| `teachers` | بيانات المدرسين (الاسم، المادة، الرسوم، نسبة المعهد) |
| `student_teacher` | جدول ربط الطلاب بالمدرسين (علاقة متعددة لمتعدد) |
| `installments` | الأقساط المدفوعة (المبلغ، تاريخ الدفع، نوع القسط) |
| `teacher_withdrawals` | سحوبات المدرسين (المبلغ، تاريخ السحب) |
| `users` | حسابات المستخدمين (اسم المستخدم، كلمة المرور، الصلاحية) |

### نظام حساب الأرصدة

يعمل النظام بنسبة مئوية قابلة للتخصيص لكل مدرس (الافتراضي 30%). يتم توزيع النسبة بالتساوي على أنواع الأقساط:

```
مشاركة المعهد = (مجموع الأقسط لكل نوع × (النسبة / 2))
رصيد المدرس = إجمالي المدفوعات - مشاركة المعهد
الرصيد المتبقي = رصيد المدرس - إجمالي المسحوبات
```

## ⚙️ المتغيرات البيئية

أنشئ ملف `.env` في جذر المشروع (يمكنك نسخه من `.env.example`):

```env
# قاعدة البيانات
DATABASE_URL="file:./dev.db"

# التطبيق
NEXT_PUBLIC_APP_NAME="نظام إدارة معهد صرح البنوك"
NEXT_PUBLIC_APP_URL="http://localhost:3000"

# المصادقة (اختياري - لمفتاح الجلسة)
AUTH_SECRET="your-secret-key-here-change-in-production"
```

| المتغير | الوصف | مطلوب |
|---------|-------|-------|
| `DATABASE_URL` | مسار قاعدة بيانات SQLite | ✅ |
| `NEXT_PUBLIC_APP_NAME` | اسم التطبيق | ❌ |
| `NEXT_PUBLIC_APP_URL` | رابط التطبيق | ❌ |
| `AUTH_SECRET` | مفتاح تشفير الجلسات (الإنتاج) | ❌ |
